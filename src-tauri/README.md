# src-tauri — the Tauri v2 shell

This crate is the desktop shell: it exposes `skill-cellar-core` to the React
frontend via Tauri IPC commands and supplies the live HTTP registry fetcher.

All testable I-1 logic lives in `../core` (which builds and tests with no Tauri
or system dependencies). This crate links the native webview, so it has heavier
build prerequisites.

## Build prerequisites

- **Rust ≥ 1.77.2** (Tauri v2 MSRV). Install/upgrade with `rustup`.
- **Linux/WSL2 system libraries:**
  ```bash
  sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget file \
    libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev
  ```
  A visible window under WSL2 additionally needs WSLg (`$DISPLAY` set). If the
  window renders blank, try `WEBKIT_DISABLE_DMABUF_RENDERER=1`.
- **Icons:** run `npx tauri icon <logo.png>` once (see `icons/README.md`).

## Run

```bash
npm install          # from the repo root
npm run tauri:dev    # launches Vite + compiles Rust + opens the window
```

## Notes

- `src/api/bindings.ts` (frontend) is regenerated by tauri-specta on every debug
  build. The version checked in is a bootstrap; do not hand-edit it once the
  shell builds.
- The `specta` / `specta-typescript` / `tauri-specta` versions in `Cargo.toml`
  are pre-1.0 release candidates and must be pinned to a mutually compatible set
  on the first successful build.
- `REGISTRY_URL` in `src/lib.rs` is a placeholder; the canonical registry
  manifest location is an open design question. Until it resolves, the shop runs
  off the on-disk cache / bundled snapshot.
